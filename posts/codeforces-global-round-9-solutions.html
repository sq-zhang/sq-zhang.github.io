A. Sign Flipping

给定一个整数数组 a，长度 n 为奇数。允许对多个元素取反，需要满足：

  • 至少有 $\frac{n - 1}{2}$ 对 $a_{n + 1} - a_{i} \ge 0$
  • 至少有 $\frac{n - 1}{2}$ 对 $a_{n + 1} - a_{i} \le 0$

可以证明至少存在一种方案，输出其中一个可行解。

思考

如果往数组中间插入不等号，最终需要形成 <= 与 >= 各一半的局面。

最明显的一种解是 <= 与 >= 交替出现。如果数组允许排序的话，可以先排序，再将后半部分隔一个的插入，即可得到 “<= >= <= <=…” 的数组。但是题目是不允许排序的。（遂陷入迷茫中…）

不过允许对元素取反！所以只需按奇偶确定元素的符号即可达成 “<= >= <= <=…”！

B. Neighbor Grid

给定一个 n x m 的非负整数矩阵。定义一个完美矩阵：对矩阵中的每个数 k，当 k > 0 时，它的相邻元素正好有 k 个数大于 0。

你可以对矩阵中的任意位置的元素无限次数的加 1。相邻元素是指由共同的边的元素。

输出一个完美矩阵，如果不存在则输出 “NO”。

思考

每一个单元格的值都是有上限的，因为相邻的元素最多 4 个，如果在边界，则可能是 2 个或 3 个。所以如果超出上限，则不存在完美矩阵。

因为每一个元素可无限加 1，所以达成完美矩阵的一种解是让每个元素都加到它的上限。那样每个单元格都是符合条件的。

（如果希望得到元素和最小的完美矩阵，应该如何做呢？）

C. Element Extermination

给定长度为 n 的数组 a，它是数字 1 - n 的排列。在一次操作中，你可以选择一个 i，其中 $a_{i} < a_{i + 1}$，然后移除 $a_{i}$ 或 $a_{i + 1}$。移除后，剩余的元素会合并在一起。

问是否可能最后只剩下长度为 1 的数组。

思考

观察样例可以发现，当最后一位是 1 是，是不可能最后只剩下一个的，因为 1 是没法消除的，左边的任何一个数都不能与 1 配对，最后总会剩下大于 1 的数在左边。

推理开来，当 $a_1 > a_n$ 时，是不可能最后只剩下一个的。这种情况下，我们可以通过 $a_1$ 与中间大于 $a_1$ 配对，将它们消除，同时将中间小于 $a_n$ 的数消除，最后剩下 $(a_1, a_n)$ 是无法消除的。即使在上述步骤中，将 $a_1$ 或 $a_n$ 消除了，最后剩下最左边的元素总是会大于最右边的元素，最终都是无法消除的。

D. Replace by MEX

给定长度为 n 的数组 a（$0 \le a_i \le n$）。
在一次操作中，你可以从数组中选中一个数，用 MEX 代替。MEX 是指 [0, n] 中最小未出现在数组中的正整数。每一次替代之后，MEX 都可能改变。

你需要在最多 2n 次操作后，将数组变为非递减数组。

思考

我们发现 [0, 1, 2, …, n - 1] 是满足条件的非递减数组。

在每次操作中，如果 mex = n，也就是 [0, n - 1] 都已出现在数组中，只是某些元素位置不对，这时任意找到一个 i != a[i]，替换成 a[i] = mex；如果 mex < n，说明 mex 不存在数组中，直接将令 a[mex] = mex；

这样最多 2 次操作可以将一个元素放在正确的位置上，总共有 n 个数字，满足最多 2n 次操作的条件。

算法时间复杂度是 $O(n^2)$。

E. Inversion SwapSort

一个逆序对 (u, v)是指满足 $u < v 且 a_u > a_v$ 的数对。

给定一个数组 a，在一次操作中，你可以交换逆序对的两个数字。你需要找到一个数对列表，满足以下条件：

  1. 所有数对都是逆序对
  2. a 中的所有逆序对都在列表里
  3. 按照逆序对的顺序交换数字，最终数组是非递减数组

输出这样一个列表或判断它不存在。

输入：第一行数组长度 n （$1 \le n \le 1000$）；第二行数组 a（$1 \le a_i \le 10^9$）

输出：如果列表不存在，输出 -1；否则第一行输出 m，接着输出 m 行，每行输出两个数字代表数对下标

思考

我们先解决 a 中不存在重复元素的情况，此时 a 是 n 的一个全排列。

定义 $pos_i(1 \le i \le n)$ 表示 i 在排列中的位置。假设 a[1] != 1（如果 a[1] = 1，则去除 1，从 2 开始计算），那么必定存在逆序对 $(1, pos_{a_1 + 1}), (1, pos_{a_2 + 1}),…,(1, pos_{a_n + 1})$。

像冒泡排序那样，每轮操作的目标是将最小值交换到最前面的位置。如果按 $(1, pos_{n}), (1, pos_{n - 1}),…,(1, pos_{a_1 + 1})$ 的顺序执行交换。定义 b 为交换后的数组，那么有 $b_1 = 1$ 且 b[1:] 的逆序对与 a[1:] 的逆序对是相同的。

以 a = {4, 3, 1, 2} 为例，此时 a[1] = 4，存在 3 个逆序对 (1, 2), (1, 3), (1, 4)。将它们按 a[i] 的值逆序排序，得到 (1, 2),(1, 4), (1, 3)。执行三次交换后，得到数组 b = {1, 4, 2, 3}。可以发现 a[1:] = {3, 1, 2} 和 b[1:] = {4, 2, 3} 的逆序对是相同的。

这样从 1 到 n 完成交换后，即可得到排序好的 1 到 n。

再来考虑存在重复元素的情况。我们可以在排序时加入下标一起排序，下标大的数排在前面。这样就可以按照全排列的方式完成排序了。

最终的算法时间复杂度为$O(n^2)$。

F. Integer Game

这是一道交互题。
给定三个正整数 a, b, c 表示三堆石头（三个数各不相同）。游戏规则：在每一轮，先手选择一个整数 y，后手选择将 y 加到其中一堆石头上。注意：连续两轮不能选同一个堆。

当其中两堆具有相同的石头时，后手输了。如果玩到 1000 轮以后，先手输了。

Anton 决定让 Harris 选先手还是后手，帮助 Harris 赢得比赛。

输入：第一行输入 a，b，c 三个数（$1 \le a,b,c \le 10^9$）

交互：在读取三个数字后，打印 “First” 或 “Second”。在每轮中，先手输出数字 y（$1 \le y \le 10^12$），后手输出 1，2 或 3 表示将 y 加到第几堆。在连续两轮中，必须选不同的堆。

如果你是后手且完成 1000 轮没有输，或者你是先手，交互程序无法继续，交互程序将会打印 0，代表测试通过，你需马上退出程序。

如果你是先手且完成 1000 轮，或者你是后手，无法行动时，交互程序将会打印 -1，代表测试未通过。

思考

要理解这题，必须先明白，后手是怎么会输的？难道真的需要玩 1000 轮吗？

关键点是连续两轮不能重复选同一个！

我们假设有 (a, b, c) 三个数，先手给出 $y_1$，后手任意选一个数，假设是 3，得到 $(a, b, c’ = c + y_1)$ 三个数。下一轮先手给出 $y_2$，注意此时后手只能选 1 或 2，如果 a 和 b 加上 $y_2$ 后都会重复，那么后手就输了。

所以问题的关键是如何造出这样的场景？答案是造等差数列！

不妨假设 a < b < c，令 $y_1 = 2c - a - b$，后手有三种选择：

  • 选 1，得到 (b, c, 2c - b) 的等差数列，令 $y_2 = c - b$，后手选 b 或 c，都会重复，所以后手输了。
  • 选 2，得到 (a, c, 2c - a) 的等差数列，令 $y_2 = c - a$，同样的，后手输了。
  • 选 3，得到 (a, b, 3c - a - b)。再重复 $y_1 = 2c’ - a - b$，后手无法重复选 3，只能选 1 和 2，再重复上面的操作，后手必输。

G. Tree Modification

给定一个带有 n 节点的树，允许对树的节点做下列操作：

  1. 选中三个节点 a, b 和 c，其中 b 与 a 和 c 都相连
  2. 对所有与 a 相连的其他节点（除了 b）d，删除 d 与 a 的连接，将 d 与 c 相连
  3. 删除 a 与 b 的连接，将 a 与 c 相连

输出将树改变为星型结构所需的最小操作次数。

输入：第一行树输入节点数 n（$3 \le 2 * 10^5$），接着输入 n - 1 行，每行输入树的边 $u_i$ 和 $v_i$ ($1 \le u_i,v_i \le n, u_i \ne v_i$)。

输出：最小操作次数 k

思考

注意到树的一个性质：树是一个二分图，它的节点可以被涂成白色或黑色，且白色节点只与黑色节点相连，相反也一样。而星型结构的树就是其中一种颜色只有一个节点。

假设 a，b，c 中，b 是黑色节点，那么 a 和 c 一定是白色节点， 执行一次操作后：

  • 所有与 a 相连的其他节点（除了 b）连接到 c，仍然是黑色节点
  • a 连接到 c，那么 a 要从白色变到黑色
  • 所有其他节点都不影响

所以每次操作只会改变一个节点的颜色。假设初始时有 l 个黑色节点和 r 个白色节点，那么答案即为 min(l - 1, r - 1)。

剩下的问题是，如何将树转化为二分图？

按奇偶层数二分即可，需要用到深度优先搜索。所以时间复杂度为 $(n)。
