数据结构之树状数组 Fenwick Tree

发表于 2020-07-09 分类于 数据结构 阅读次数：
本文字数： 5.9k 阅读时长 ≈ 15 分钟
树状数组 Fenwick Tree 是常用的数据结构，它在范围查询方面有较好的应用，能在 O(log n) 时间内完成范围的修改和查询。本文将介绍树状数组的工作原理、应用场景以及 OJ 例题。

A. 简介

树状数组 Fenwick Tree，是由 Peter M. Fenwick 在 1994 年发表的论文 《A new data structure for cumulative frequency tables》中提出的一种数据结构。

它的应用场景是数组，假设长度为 n，它有以下特性：

  1. O(n) 时间内构造树状数组
  2. O(log n) 时间内计算 [l, r] 范围内的 f 值（f 是一种函数，比如 sum，max，min 等）
  3. O(log n) 时间内更新数组的值
  4. 需要 O(N) 存储空间
  5. 实现简单，场景可拓宽到多维数组

B. 原理

树状数组的基本思想是分治，用一个节点表示多个节点的值，那样在查询时只需查询大节点，而不用查询每个节点。

比如，在上图中，数组 tree 是一个表示范围和的树状数组。在该数组中：

  • tree[8] 代表全部 8 个数
  • tree[6] 代表 a[5] 和 a[6]
  • tree[4] 代表 a[1]、a[2]、a[3]、a[4]
  • tree[2] 代表 a[1]、a[2]
  • tree[1]、tree[3]、tree[5]、tree[7] 只代表本身

细心的你可以发现，对于 i，假设它的二进制表示中最右一个 1 的位置为 p，那么 tree[i] 代表的个数为 $2^p$（所以 tree 的下标是从 1 开始的，因为 0 不存在二进制为 1 的位）。比如

  • i = 3(101)，最右一个 1 位于第 0 位，tree[3] 代表一个数，也就是 a[3] 本身。
  • i = 4(100)，最右一个 1 位于第 2 位，tree[4] 代表 4 个数，也就是 a[1]、a[2]、a[3]、a[4]。

i & (-i) = i 代表的个数 $2^p$。

应用

场景一：单点修改，区间查询

场景：多次修改单点，求范围和

初始化 O(n)

从树状数组的定义可知，每一个节点与子节点的值直接相关。因此可以从底向上更新父节点的值。i 的父节点为 i + i & (-i)。

范围查询 O(log n)

范围查询包括前缀和和范围和。范围和可通过前缀和计算得到。

对于前缀和，根据定义，计算 i 的前缀和的过程也是遍历 i -> 1 的过程，原始的遍历计算是 O(n) 的时间，使用树状数组的遍历时间为 O(log n)。

以 i = 6 为例，遍历的第一个节点是 tree[6]，它代表 5，6 两个数，因此下一个节点是 tree[4]，它代表 4 个数，到达 tree[0] 结束。因此 i = 6 的前缀和为 tree[6] + tree[4]。

假设查询的范围为 [left ,right]，那么 $$sum(left, right) = prefixSum(right) - prefixSum(left - 1)$$

单点修改 O(log n)

单点修改需要递归地修改它的父节点。

场景二：区间修改，单点查询

场景：多次区间修改，求单点值

通过将树状数组定义为每个元素与前一个元素的差，那么它的前缀和和就等于原数组。这样一来，单点查询转变为范围查询，这与场景一相同。区间修改可通过修改树状数组的两个单点实现。所以场景二本质上与场景一相同，只是树状数组定义不同。

初始化 O(n)

构建树状数组的过程与场景一相同，初始值改为每个元素与前一个元素的差值。
设原数组为 a[i]，那么树状数组 tree[i] = a[i] - a[i - 1](a[0] = 0)。

单点查询 O(log n)

改变树状数组的定义后，单点查询等同于场景一的范围查询，代码见场景一的 prefixSum 函数。
$$a[i] = \sum_{j = 1}^{i} tree[j]$$

范围修改 O(log n)

假设要给区间 [l, r] 加上 x，根据树状数组的定义，只需给 tree[l] 加上 x，给 tree[r + 1] 减去 x。因为给 tree[l] 加上 x ，相当于给所有的 a[i] (i >= l) 都加上了 x。再给 tree[r + 1] 减去 x，把对 i > r 的影响消除，从而实现了给区间 [l, r] 加上 x。

场景三：区间修改，区间查询

场景：多次区间修改，求范围和

场景三是基于场景二来完成的，同样将树状数组定义为每个元素与前一个元素的差。那么前缀和就可表示为 $$sum(p) = \sum_{i = 1}^{p} \sum_{j = 1}^{i} tree[j]$$

观察可以发现，tree[j] 被加上的次数等于 p - i + 1。所以前缀和还可以表示为 $$sum(p) = \sum_{i = 1}^{p} tree[i] * (p - i + 1) = (p + 1) * \sum_{i = 1}^{p} tree[i] - \sum_{i = 1}^{p} tree[i] * i$$

惊喜的看到，前缀和可以由另外两个数组的前缀和表示。
所以，可以通过维护 tree[i] 与 tree[i] * i 的树状数组来求范围和。

初始化 O(n)
根据上面的分析，我们要维护 tree[i] 与 tree[i] * i 的树状数组，树状数组的初始化都是一样的, 要注意的是 tree2[i] = tree1[i] * i。

范围查询 O(log n)
位置 p 的前缀和即等于 (p + 1) * tree1 的前缀和 - tree2 的前缀和。

范围修改 O(log n)
范围修改同场景二，同样基于单点修改。单点修改时要同时修改 tree1 和 tree2。

C. 推广到二维数组

在一维数组中，tree[i] 表示的是长度为 lowbit(i)，终点为 i 的范围和。那么，在二维数组中，tree[x][y] 表示长为 lowbit(x)，宽为 lowbit(y)，右下角为 (x, y) 的矩阵和。

相应的二维树状数组的单点修改和区间查询都需要加上一个维度。

在范围修改时，同样需要将树状数组定义为每个元素与前一个元素的差。在一维数组中，sum[i] = sum[i - 1] + a[i], tree[i] = a[i] - a[i - 1]。那么，在二维数组中 tree[x][y] 该如何定义呢？

这与前缀和的定义有关，如图所示，二维数组的前缀和 sum[x][y] 与 sum[x - 1][y - 1] 的关系表示为 $$ sum[x, y] = sum[x - 1][y] + sum[x][y - 1] - sum[x - 1][y - 1] + a[x][y]$$
对应地，tree[x][y] 即可表示为$$ tree[x][y] = a[x][y] - a[x - 1][y] - a[x][y - 1] + a[x - 1][y - 1]$$

跟一维数组一样，对原数组的单点查询即为该点的前缀和。

那剩下的问题是如何做范围修改？

假设要在 (x, y) 和 (m, n) （假设 $m \ge x, y \ge n$） 之间加上一个数 z。如果在 tree[x][y] 处加上 z，它影响的范围是 $i \ge x || j \ge y$ 的部分(图中标号为 1)。

  • tree[x][y] + z 影响图中标号为 1 的部分
  • tree[x][n + 1] - z 影响图中标号 2 的部分
  • tree[m + 1][y] - z 影响图中标号为 3 的部分
  • tree[m + 1][n + 1] + z 影响图中标号为 4 的部分

为了消除对 $i > m || j > n$ 的影响（这部分超出了我们的逾期），我们需要在 tree[x][n + 1] 和 tree[m + 1][y] 处减去 z，再在 tree[m + 1][n + 1] 处加上 z 以补偿减了两次的 i > m && j > n 区域。

范围修改配合范围查询在二维数组下更加复杂了。定义二维数组的前缀和为 sum(x, y)，那么有
$$sum(x, y) = \sum_{i = 1}^{x}\sum_{j = 1}^{y}\sum_{k = 1}^{i}\sum_{h = 1}^{j} tree[k][h]$$

同一维数组时一样，将它展开可以得到：
$$sum(x, y) = (x + 1) * (y + 1) * \sum_{i = 1}^{x}\sum_{j = 1}^{y} tree[i][j] - (y + 1) * \sum_{i = 1}^{x}\sum_{j = 1}^{y} tree[i][j] * i - (x + 1) * \sum_{i = 1}^{x}\sum_{j = 1}^{y} tree[i][j] * j + \sum_{i = 1}^{x}\sum_{j = 1}^{y} tree[i][j] * i * j$$

因此，我们可以维护四个树状数组：tree[i][j],tree[i][j] * i,tree[i][j] * j, tree[i][j] * i * j。

二维数组的完整代码见 Github 仓库。
