Atcode Beginer Contest 173 题解

Atcode Beginer Contest 173 题解，共 6 题，A-B 是送分题，C 涉及状态的压缩，D 是 K 数相乘，F 涉及图论连通分量的计算

A - Payment

我们要买价值 n 元的产品，如果我们只用 1000 面值的纸币。
输出得到的找零。

思考

为初次参赛准备的入门题。
明显答案为 1000 - n % 1000。当 n 为 1000 的整数倍时，输出 0。

B - Judge Status Summary

给定整数 n，以及 n 个评分状态。统计评分状态中 AC、WA、TLE 和 RE 出现的次数。

思考

另一道为初次参赛准备的入门题。
可以使用哈希表统计出现字符串的次数，或者直接只统计这 4 个状态。

C - H and V

给定 H x W 的单元格矩阵 c，矩阵的元素 $c_{i,j}$ 有两种取值：’.’ 或 ‘#’，其中 ‘.’ 表示单元格为白色，’#’ 表示单元格为黑色。
你可以选择任意行或任意列，使得整行或整列变为红色。
给定一个整数 K，问有多少种涂色方案使得矩阵中正好剩下 k 个黑色单元格。

数据要求

  • $1 \le H,W \le 6$
  • $1 \le K \le H*W$
  • $c_{i,j}$ 为 ‘.’ 或 ‘#’

思考

首先要充分理解题意，行和列的个数是任选的，也可以是 0。

一种想法是，统计每行每列黑色的个数，然后任意组合得到被覆盖的黑色的个数。当 $c_{i,j}$ = ‘#’ 时，同时选中第 i 行和第 j 列要避免重复计算的问题。

另一种想法是，遍历状态，每行和每列都有选中和不选中两种状态，状态总数为 $2^{H+W}$。因为 $1 \le H,W \le 6$，所以不必担心会超时。而且可以用比特位表示状态，比如 6 = 0110，表示选中第 2 行和第 3 行。在每种状态下统计 ‘#’ 的个数，恰好等于 k 则找到一种可行解。

D - Chat in a Circle

给定 n 位选手，编号从 1 到 n。给定一个数据 A，其中 $A_i$ 表示 i 号选手的友好度。
选手们依次到场，而且围成一个圈。当一名选手加入圈内，它会获得左右两边选手中友好度较小的一个。第一名选手得到 0。
输出 n 名选手总共能得到的最大友好度。
数据要求

  • $2 \le n \le 2 x 10^5$
  • $1 \le A_i \le 10^9$

思考

贪心算法：欲使得到的友好度最大，必须按友好度降序的顺序安排选手。由于选手围成一个圈，所以每次能得到圈中最大友好度最多 2 次，最大值只能获得 1 次。

以 9，8，7，6，5 为例，先往圈中放入 9。之后放入 8，友好度增加 9；放入 7 时，有两个位置可选，都只能友好度加 8；放入 6 时，放在 8、9 中间，友好度加 8；同理，5 为了避免在 6 身边，最多只能得到 7，以此类推。

所以答案为 $\sum_{k=1}^{n - 1} A_{n - k / 2}$。
算法时间复杂度为 $O(n\log{n})$，瓶颈在排序上。也可以通过将排序降为 O(n)，得到 O(n) 的解法。

E - Multiplication 4

给定 n 个整数，从中选择 k 个数相乘。
输出最大乘积，结果取模 1e9 + 7。
数据要求

  • $1 \le k \le n \le 2 x 10^5$
  • $|A_i| \le 10^9$

思考

显然，当 n = k 时，或所有数都是负数，且 k 是奇数时，最佳选择是选 k 个绝对值最小的数相乘。
其他情况，答案都是非负数，因此只需尽量将绝对值最大化。

绝对值最大化的一种方案是将 A 按绝对值降序排序，取前 k 个数相乘，如果得到非负数，那么它就是答案。否则需要将它变为非负数，这时有两个选项：

  1. 移除绝对值最小的负数，加入决定值最大的非负数
  2. 移除绝对值最小的正数，加入绝对值最大的负数

另一种方案是将正负分开，分别按顺序两两配对，贪心的从中选取乘积大的加入。

（如果 0 的数量过多，可提前判断，优化速度）

F - Intervals on Tree

给定带 n 个节点和 n - 1 条边的树。令 S 为节点编号从 L 到 R 的节点集合。定义 f(L, R) 等于所有 S 中的节点到 S 中的节点的连接数。

输出 $\sum_{L=1}^{N} \sum_{R=L}^{N} f(L,R), (1 \le L \le R \le n)$

数据要求

  • $1 \le n \le 2 * 10^5$
  • $1 \le u_i,v_i \le n$

思考

解法一

利用并查集计算每个 L,R 对应的连通分量，累加得到答案。
时间复杂度是 $O(n^2 \alpha(n))$

解法二

考虑树的一个性质：节点数 = 边数 + 1。而在 S 内形成的连通分量都是一棵树，所以有连通分量 $m = S_{节点数} - S_{边数}$。

现在问题转化为在所有的 S 中，包含多少个节点？包含多少条边？它们的差就是答案。

假设 n = 5，对节点 i = 4，什么样的 L，R 能使 S 包含 i = 4？

显然，L 的取值范围为 [1, 4]，R 的取值范围为 [4, 5]，所以包含 i = 4 的 S 的个数为 4 * 2 = 8，也就是 $i * (n - i + 1)$。

接着来看边的情况。假设有一条边 e = (4, 2)。S 要包含这边边，必须同时包含节点 2 和 节点 4。那么 L 的取值范围为 [1, 2]，R 的取值范围为 [4, 5]，所以包含 e 的 S 的个数为 2 * 2 = 4，也就是 $min(u_i, v_i) * (n - max(u_i, v_i) + 1)$。

算法时间复杂度为 O(n)。
